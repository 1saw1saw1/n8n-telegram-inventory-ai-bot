Inventory & AI Analytics Bot
🚀 Описание проекта
Интеллектуальная система автоматизации складского учета, обеспечивающая синхронизацию данных между CRM и базой данных Supabase. Система использует LLM для анализа товарных остатков и обработки запросов пользователей через Telegram-интерфейс, позволяя принимать решения на основе данных в режиме реального времени.

🛠 Технические особенности
Архитектура построена на принципах Event-Driven Automation с использованием локального инстанса n8n и интеграции с внешними сервисами через REST API:

Telegram Bot Integration: Полный цикл взаимодействия с пользователем через API Telegram.

n8n Workflow Automation: Центральный узел управления всей логикой сценариев.

Supabase (HTTP REST API): Хранение сессий и данных через прямое взаимодействие с API (вместо использования нативных узлов БД, что обеспечивает легкость и переносимость).

FMS (Flow Management System): Реализованная на уровне n8n система управления состоянием пользователя (state management) для контекстного диалога.

Dynamic User Session Tracking: Отслеживание состояний пользователя в реальном времени.

JavaScript: Реализация сложной бизнес-логики и трансформации данных непосредственно внутри воркфлоу. Использование JS для нормализации JSON-ответов от API, реализации условной фильтрации и подготовки данных для LLM-инференса. 

Product Search & Analytics: Потоковая обработка запросов с использованием AI для глубокого анализа товаров.

Scheduled Inventory Analysis: Автоматизированные задачи по расписанию для мониторинга остатков.

REST API Integration: Взаимодействие с внешними CRM и аналитическими системами для обмена данными.

📋 Как развернуть проект
Предварительные требования
Установленный инстанс n8n.

Аккаунт в Supabase и Groq.

Токен бота Telegram (получить у @BotFather).

Шаги по настройке
Импорт: Скопируйте содержимое supabaseclean.json и вставьте в редактор n8n (нажмите Ctrl+V на пустом поле).

Настройка Credentials:

Создайте новые учетные данные для Telegram API (вставьте Bot Token).

В узлах HTTP Request замените YOUR_CRM_API_KEY_HERE на ваш ключ доступа к CRM.

В узлах Supabase замените YOUR_SUPABASE_KEY_HERE на ваши ключи API из панели управления Supabase.

Примечание: Для работы с Supabase/PostgreSQL в данном проекте используются HTTP-запросы (REST API), что позволяет запускать воркфлоу на любой инфраструктуре без необходимости установки специфических драйверов БД.

🔐 Безопасность
Для обеспечения безопасности в публичном доступе в предоставленном файле supabaseclean.json реальные ключи доступа были удалены и заменены заглушками YOUR_..._HERE.

Проект разработан для автоматизации ритейл-процессов и оптимизации операционной эффективности.
